## Storage Class Notes

The storage classes all expose public fields and have constructors that accept `ModElement`.

Important limitation: constructor defaults are incomplete. Many public fields remain `null` after construction, especially plugin-specific references such as models, textures, procedures, sounds, and several element settings. This means fully valid automated creation cannot be assumed just because a storage class can be instantiated.

Observed constructor defaults:

- `AnimatedItem`
  - Sets `rarity = "COMMON"`, `inventorySize = 9`, `inventoryStackSize = 64`, `saturation = 0.30000001192092896`, `animation = "eat"`, `armPoseList = []`, `disableSwing = false`, `creativeTabs = []`.
  - Leaves fields such as `texture`, `customModelName`, `name`, `idle`, `normal`, and procedures unset.
- `AnimatedEntity`
  - Sets defaults such as `modelShadowSize = 0.5`, `mobCreatureType = "UNDEFINED"`, `mobModelTexture = ""`, `trackingRange = 64`, `rangedItemType = "Default item"`, `rangedAttackInterval = 20`, `rangedAttackRadius = 10.0`, `followRange = 16`, `inventorySize = 9`, `inventoryStackSize = 64`, `stepHeight = 0.6`, `entityDataEntries = []`, `raidSpawnsCount`, `creativeTabs = []`, and `attackRate = 7`.
  - Leaves many required gameplay, AI, model, texture, and animation fields unset.
- `AnimatedBlock`
  - Sets defaults such as `blockstateList = []`, `tintType = "No tint"`, `boundingBoxes = []`, `restrictionBiomes = []`, `blocksToReplace = []`, `reactionToPushing = "NORMAL"`, `slipperiness = 0.6`, `speedFactor = 1.0`, `jumpFactor = 1.0`, `colorOnMap = "DEFAULT"`, `aiPathNodeType = "DEFAULT"`, `offsetType = "NONE"`, `generationShape = "UNIFORM"`, inventory/energy/fluid lists and defaults, `animationCount = 1`, `vanillaToolTier = "NONE"`, `creativeTabs = []`.
  - Leaves textures, block name, base block, material, sounds, and procedures unset.
- `AnimatedArmor`
  - Sets item render types to `0`, item custom model names to `"Normal"`, special info lists to `[]`, and `creativeTabs = []`.
  - Leaves armor model, armor texture, piece names, item textures, enabled pieces, repair items, equip sound, and procedures unset.

MVP implication: `createGeckoLibElement` must validate conservatively. If a complete and reload-safe object cannot be created through MCreator APIs, implement a shell creation path or return a clear error. Do not fake unknown plugin fields.

## Confirmed Asset Directories

### Runtime paths generated by Nerdy's GeckoLib Plugin

The plugin templates reference runtime resources under the generated mod assets:

- Geo models: `assets/<modid>/geo/<model>.geo.json`
- Animations: `assets/<modid>/animations/<model>.animation.json`
- Animated item textures: `assets/<modid>/textures/item/<texture>.png`
- Animated entity textures: `assets/<modid>/textures/entities/<texture>.png`
- Animated block textures: `assets/<modid>/textures/block/<texture>.png`
- Animated armor texture path: `assets/<modid>/textures/item/<armorTextureFile>` according to `aarmor/armor_model.java.ftl`

### MCreator workspace source paths

This is the plan-changing mismatch.

MCreator 2024.4 does not generally manage editable workspace resources directly under `src/main/resources/assets/<modid>/...`. The confirmed workspace-side resource locations are:

- Models directory: `WorkspaceFolderManager#getModelsDir()` -> `<workspace>/models/`
- Model animations directory: `WorkspaceFolderManager#getModelAnimationsDir()` -> `<workspace>/models/animations`
- Texture directories: `WorkspaceFolderManager#getTexturesFolder(TextureType)` using generator keys:
  - `item_textures_dir`
  - `block_textures_dir`
  - `entity_textures_dir`
  - `armor_textures_dir`
  - and other `TextureType` IDs

The import plan should therefore target MCreator workspace resource directories first, not `src/main/resources` directly. Generation then copies or references resources into runtime asset paths.

## Safe Public GeckoLib Plugin API

No safe public API for MCP automation was found. The plugin exposes Java classes, actions, UI panels, and element storage classes, but the MVP should not compile against them.

Allowed interaction model:

- Generic MCreator APIs.
- Reflection only after confirming `geckolib_plugin` is loaded.
- Filesystem inspection for assets.
- Defensive serialization/deserialization through MCreator `ModElementManager`.

Forbidden interaction model:

- Static imports from `net.nerdypuzzle.geckolib...`
- Static imports from any class with `geckolib`, `nerdy`, or animated element package names.
- Assumptions that constructor defaults create a fully valid element.

## Risks and Limitations

- Upstream commit SHA for the inspected source was not confirmed due blocked network access. The local zip hash is the exact inspected artifact.
- The plugin is archived and described publicly as abandoned, so future fixes should not be assumed.
- Existing plugin comments and templates indicate users must enable the GeckoLib API manually; MCP must not silently change this.
- Creating complete GeckoLib elements without the MCreator UI is risky because plugin fields are broad, public, and not fully initialized by constructors.
- Direct JSON schema generation is not reliable for this plugin. The MVP should not expose `workspace://geckolib/schema`.
- Updating existing GeckoLib elements is unsafe for MVP. The MVP should not implement `updateGeckoLibElement`.
- Asset import must be transactional. Because workspace resource directories differ from runtime paths, the implementation must first resolve the correct workspace destinations before copying.

## Recommended Phase 1 Adjustment

Before implementing tools, update the implementation plan as follows:

- Keep `getGeckoLibStatus` and `workspace://geckolib/status` as planned.
- Implement `listGeckoLibAssets` against MCreator workspace resource directories:
  - `models/` for `.geo.json`
  - `models/animations/` for animation resources only if the plugin/MCreator importer stores GeckoLib `.animation.json` there, otherwise document as unsupported until import behavior is confirmed in a live workspace
  - generator texture directories for item, block, entity, and armor textures
- Do not hardcode `src/main/resources/assets/<modid>/geo` as the source workspace import destination.
- For `createGeckoLibElement`, start with `animateditem` only if a reload-safe minimal object can be proven through MCreator serialization and reopen testing. Otherwise expose it as shell creation or return a clear unsupported message.
